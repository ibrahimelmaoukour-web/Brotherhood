# ❓ Frequently Asked Questions\n\n## 🎯 About Brotherhood\n\n### What is Brotherhood?\n\nBrotherhood is a free, open-source, community-driven reference for primitive crafting techniques. It documents the progression of human civilization through crafts — from discovering water to building electricity systems.\n\n**Key Features:**\n- 📚 Comprehensive craft roadmaps\n- 🌍 Open to all contributors\n- 🚀 Zero frameworks, pure vanilla code\n- 🏠 Works entirely in the browser\n- 💾 Data saved locally\n\n### Who created Brotherhood?\n\nIbrahim Elmaoukour created the initial concept and app architecture. It is now maintained by the community!\n\n### Is Brotherhood free?\n\nYes! 100% free and open-source under the MIT License. You can use, modify, and share it without restrictions.\n\n---\n\n## 🚀 Getting Started\n\n### How do I use Brotherhood?\n\n1. Clone or download the repository\n2. Open `bortherhood/index.html` in your browser\n3. Authenticate with password: `FORGE`\n4. Browse crafts and learn!\n\n### Do I need to install anything?\n\nNo! Brotherhood uses vanilla HTML/CSS/JavaScript. Just open the HTML file in your browser. No build tools, Node.js, or dependencies needed.\n\n### What browsers does it work in?\n\nBrotherhood works in all modern browsers:\n- Google Chrome/Chromium\n- Firefox\n- Safari\n- Edge\n\n### Can I use it offline?\n\nYes! Once loaded, Brotherhood works entirely offline. Data is stored in browser LocalStorage.\n\n---\n\n## 🤝 Contributing\n\n### I want to contribute. Where do I start?\n\n1. **Read** [CONTRIBUTING.md](CONTRIBUTING.md)\n2. **Choose** what to contribute:\n   - New craft\n   - Craft expansion\n   - Bug fix\n   - Documentation\n3. **Follow** the guidelines\n4. **Submit** a pull request\n\n### Do I need programming experience?\n\nNo! Most contributions are writing crafts and documentation. You need:\n- Basic understanding of the format (copy existing examples)\n- Ability to test in a browser\n- Attention to detail\n\nNo coding skills required!\n\n### What can I contribute?\n\nAnything helps!\n\n✅ **High Priority:**\n- New craft roadmaps (choose from ROADMAP.md)\n- Completeness for existing crafts\n- Accuracy verification\n\n✅ **Also Welcome:**\n- Documentation improvements\n- Typo fixes\n- Better examples\n- Resource links\n- Suggestions and ideas\n\n### How do I add a craft?\n\n1. Read the [craft template](CONTRIBUTING.md#step-2-create-your-craft)\n2. Copy the format from existing crafts in `script.js`\n3. Fill in your craft details\n4. Test in browser\n5. Submit a pull request\n\n**Time needed:** 30 minutes to 2 hours depending on complexity.\n\n### What format should my craft use?\n\nFollow the existing format in `script.js`. Each craft has:\n- **ID**: Unique identifier (lowercase with underscores)\n- **Name**: Display name\n- **Icon**: Single emoji\n- **Description**: Brief overview\n- **Stages**: 1-5 progression levels\n\nSee [CONTRIBUTING.md](CONTRIBUTING.md) for full template.\n\n### Can I edit someone else's craft?\n\nYes! We encourage collaboration:\n- Small fixes (typos, clarity) — just submit PR\n- Major changes — discuss in issue first\n- Always maintain quality and accuracy\n\n### How long does approval take?\n\nTypically **3-7 days**. We review carefully but move quickly.\n\n### What if my PR gets rejected?\n\nIt's feedback, not failure! We'll explain:\n- Accuracy issues\n- Clarity problems\n- Format inconsistencies\n- Missing details\n\nWork with reviewers to improve and resubmit.\n\n---\n\n## 💡 Contribution Ideas\n\n### I want to add a completely new craft. What should I pick?\n\nCheck [ROADMAP.md](ROADMAP.md) for a list of needed crafts:\n- Core Survival Crafts\n- Advanced Crafts\n- Specialized Knowledge\n- Interdisciplinary Topics\n\nChoose one that interests you!\n\n### Can I add modern crafts like coding?\n\nSure! As long as you can connect it to the 5 civilization stages. For example:\n- Stage 1: Basic logic circuits\n- Stage 2: Binary systems\n- Stage 3: Simple programming\n- Etc.\n\n### Can I add alternative history paths?\n\nYes! Brotherhood is flexible. Different civilizations took different paths. Document your reasoning.\n\n### What if two people work on the same craft?\n\n1. First PR gets merged\n2. Later PRs can improve/expand what's there\n3. Both contributors get credited\n4. Collaboration is encouraged!\n\n---\n\n## 🛠️ Technical Questions\n\n### Why vanilla JavaScript and no frameworks?\n\nBrotherhood prioritizes:\n- **Accessibility**: Works everywhere\n- **Performance**: No overhead\n- **Learning**: Pure code, no abstraction\n- **Independence**: No dependency management\n- **Longevity**: Won't break with library updates\n\n### How is data stored?\n\nBrotherhood uses **Browser LocalStorage**:\n- User authentication\n- Preferences\n- Community uploads (future)\n- No server needed\n\n### Can I deploy Brotherhood to my own server?\n\nYes! It's open-source MIT licensed. You can:\n- Deploy to any server\n- Host your own version\n- Modify and customize\n- Just maintain attribution\n\n### How do I modify the code?\n\n1. Edit `bortherhood/script.js` for craft data\n2. Edit `bortherhood/style.css` for styling\n3. Edit `bortherhood/index.html` for structure\n4. Test in browser\n5. Submit PR if you want to contribute back\n\n### Can I add custom crafts just for myself?\n\nYes! Edit `script.js` locally and add crafts to your copy. Just don't submit those as PRs unless they're good for everyone.\n\n---\n\n## 📋 Code Contributions\n\n### Can I add features like voting or comments?\n\nAbsolutely!
